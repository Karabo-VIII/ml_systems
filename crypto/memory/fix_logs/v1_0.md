## 2026-03-18: DIRECT_RETURN_WEIGHT fix
**Severity:** Critical (invariant violation)
**Files:** `src/wm/v1/v1_0_training/settings.py`
**Problem:** DIRECT_RETURN_WEIGHT=3.0, should be 1.0 per cross-version invariant.
>1.0 creates Huber shortcut bypassing TwoHot bottleneck (confirmed in V3/V4/V6).
**Fix:** Changed to 1.0.
**Note:** Existing V1.0 checkpoint (ShIC=0.030) was trained with DRW=3.0. V1 Transformer
architecture is less susceptible to this shortcut than Mamba/WaveNet, which is why it
passed gates despite the bug. Retraining needed to match invariant.
