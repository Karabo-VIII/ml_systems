# V4.0 Fix Log

## 2026-03-16: Temporal memorization hyperparameter fix
**Severity:** Critical (gate failure)
**Files:** `src/wm/v4/v4_training/settings.py`
**Problem:** V4 failed ShIC gate (0.0167 -> 0.0135, ShIC/IC=0.245 < 0.3). Four compounding causes:
1. RSSM latent 32x32 (1024D) vs V1.0's 24x24 (576D) = 78% more memorization capacity
2. weight_decay=1e-2 vs V1.0's 5e-2 = 5x less L2 regularization on 1.2M more params
3. batch_size=48 vs V1.0's 32 = less gradient noise (project invariant violation)
4. dropout=0.10 vs V1.0's 0.15, LR=3e-4 vs 2e-4

**Fixes applied:**
- `RSSM_LATENT_DIM`: 32 -> 24, `RSSM_CLASSES`: 32 -> 24 (FLAT_DIM: 1024 -> 576)
- `WM_WEIGHT_DECAY`: 1e-2 -> 5e-2
- `WM_BATCH_SIZE`: 48 -> 32
- `WM_DROPOUT`: 0.10 -> 0.15
- `WM_LR`: 3e-4 -> 2e-4
- `DIRECT_RETURN_WEIGHT`: 3.0 -> 1.0 (Huber shortcut + Mamba temporal memory)
- `KL_ANNEAL_EPOCHS`: 20 -> 40 (longer warmup for Mamba)
- `TEMPORAL_CTX_DROP`: 0.15 -> 0.25 (stronger ATME for more expressive architecture)

**Root cause insight:** Mamba's selective scan is strictly more expressive than causal attention for temporal patterns (input-dependent gating, no softmax competition). Needs proportionally stronger regularization.

## 2026-03-17: ATME z_post leakage fix (Mamba-specific)
**Severity:** Critical (architectural bug — ATME mechanism was a no-op)
**Files:** 16 files across all V4 variants (world_model.py, ncl_model.py, settings.py, components.py)

### CRITICAL-1: ATME temporal leakage through posterior
**Files:** `world_model.py:196-211` (all V4 variants + all ncl_model.py)
**Problem:** ATME zeros h_seq in feat_heads, but z_post = f(posterior(h_seq, obs_seq)) already
carries all of Mamba's temporal encoding. Zeroing h_seq after the posterior is cosmetic — temporal
info has already leaked. This is why V4 got ShIC=0.006 at epoch 10 (vs V1's 0.025) despite
matching V1 hyperparameters. ATME was effectively disabled for Mamba.
**Fix (two-part):**
1. When ATME fires (40% of steps): compute obs-only posterior (no h_seq) for return/regime heads
2. Normal mode: use h_seq.detach() — Mamba learns from recon/KL only, return heads READ
   temporal features but can't OPTIMIZE Mamba for memorization (analogous to JEPA frozen encoder)

### HIGH-1: ATME strength too weak (now that it actually works)
**Files:** `settings.py` (all V4 variants)
- `TEMPORAL_CTX_DROP`: 0.25 -> 0.40
- `SEQ_SHUFFLE_PROB`: 0.20 -> 0.30

### HIGH-2: TwoHotSymlog default params [-5,5] instead of [-1,1]
**Files:** `components.py` (all V4 variants)
**Fix:** Changed default params to `min_val=-1.0, max_val=1.0`

### Stale comments fixed
- ATME comment: "Zero h_seq in return/regime heads" -> "Obs-only posterior for return/regime heads"
- components.py docstrings: "Bins scaled to [-2, 2]" -> "255 bins in [-1, 1]"

### Cross-cutting note
Same ATME leakage pattern exists in V3, V5, V8 world_model.py and ncl_model.py (33 files total).
Less severe for non-Mamba architectures (Transformer/GRU encode less temporal info into h_seq)
but should be fixed when those versions are trained.

## 2026-03-18: V4 variants steps-per-epoch fix
**Severity:** Critical (undertraining -> gate failure)
**Files:** `src/wm/v4/v4_1_training/settings.py`, `src/wm/v4/v4_2_training/settings.py`, `src/wm/v4/v4_3_training/settings.py`
**Problem:** V4.1/V4.2/V4.3 had WM_STEPS_PER_EPOCH=500 and DIVERSITY_STEPS_PER_EPOCH=500.
Same pattern as V3/V6 bug. 4x less data exposure per epoch = ShIC checks fire before learning.
**Fix:** Both changed to 2000 in all 3 variants.
**Note:** V4.base already had 2000 (was fixed in prior audit). Only variants were missed.
