Runner declared V9-clean as `[13, 25, 34]`. V9 lives in 41-feat SOTA taxonomy, no f25 slice. Fixed to `[13, 34]`.

### Bug 4: Validation gate false-fail on clean (a85c4f0)
`check_gate()` at line 281-290 applied `rec > GATE_REC_MSE_MAX`, `kl < GATE_KL_MIN`, `kl > GATE_KL_MAX`. MoECleanModel has NO RSSM and NO recon decoder. Same false-fail pattern as V3 Bug 4. Fix uses `_pre_args.clean` to skip rec/kl gates on the clean variant.

## 2026-03-19: NaN collapse fix (3 changes)
**Severity:** Critical (training died at epoch 6, model weights corrupted permanently)

### NaN-1: No NaN recovery mechanism
**File:** `train_world_model.py`
**Problem:** When epoch hit max_nan_per_epoch, code just broke the step loop but model weights were already corrupted. All subsequent epochs produced NaN.
**Fix:** Added NaN recovery: reload model+EMA from last checkpoint, reinitialize optimizer+scaler to clear corrupted momentum, halve LR multiplier. Max 5 recoveries before fatal stop.

### NaN-2: LR=2e-4 too high for 9.1M param MoE
**File:** `settings.py`
**Problem:** Peak LR=2e-4 combined with Kendall w_r1=7.4x amplification caused gradient overflow from epoch 1 (GN:nan). Epoch 5 (warmup complete, full LR) triggered cascading NaN.
**Fix:** WM_LR=1e-4, WM_WARMUP_EPOCHS=10 (was 5).

### NaN-3: ATME too weak (TEMPORAL_CTX_DROP=0.15)
**File:** `settings.py`
**Problem:** CLAUDE.md invariant requires 0.40 for RSSM models. Missed in V9 audit.
**Fix:** TEMPORAL_CTX_DROP=0.40, SEQ_SHUFFLE_PROB=0.30.
