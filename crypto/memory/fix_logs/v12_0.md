# V12.0 Fix Log

## 2026-04-08: VIB bottleneck added + dead-code finding (commit 9973ca0)
**Severity:** HIGH (memorization risk; was blocked from f34 tournament)

### Bug: Same memorization bug class as V3-clean
**Files:** `world_model.py`, `settings.py`

`get_loss → forward_train → forward_single_asset → WaveNet → return_head` is the actual training path. The `forward_multi_asset` (cross-asset attention innovation) is **DEAD CODE** in the standard runner — it requires synchronized multi-asset batches that the standard `AntifragileDataset` does not provide.

This means V12 trains as `LightWaveNet → direct head`, which is the same architecture as V3-clean and has the same memorization vulnerability (per Pattern I in fix_logs/INDEX.md).

**Fix:** Added Variational Information Bottleneck on `forward_train`:
- VIB_Z_DIM=16 (smaller because d_model=128, not 256)
- VIB_KL_WEIGHT=0.05, annealed via VIB_KL_ANNEAL_EPOCHS=20
- Return heads now read `z_expand(z)`, regime head still reads h_seq

**Smoke:** 847,172 params, KL=0.0008 at init, grad flows on to_mu (2.52).

### Open structural issue (not in this fix)
V12's headline innovation is cross-asset attention, but the standard runner doesn't exercise it. Two options for a future session:
1. Implement multi-asset batching in the dataloader and switch get_loss to forward_multi_asset
2. Document V12 as "single-asset baseline + dead cross-attention path" and scope the cross-attention to a separate evaluation script

Until then, V12 is effectively V3-clean-with-LightWaveNet. The VIB fix gives it the same defense.
