# V1.1 Fix Log

## 2026-06-12: Native C++ abort at epoch boundary = GPU VRAM CONTENTION (not a code bug)
**Severity:** High (run-killing crash; NOT data/checkpoint loss)
**Symptom:** `vsn_fr` run crashed at epoch 31 step 0 with `Unhandled exception caught in c10/util/AbortHandler.h`
-> `terminate` / `CxxThrowException` (a native C++ abort -- UNCATCHABLE by Python try/except), immediately AFTER
saving a strong epoch-30 checkpoint (ShIC 0.0278, IC1 0.0568, GATE PASS).
**Root cause:** NOT a trainer bug. The 8GB RTX 4060 was OVERSUBSCRIBED -- the WM training shared VRAM with the
other instance's Atari AlphaZero training (`az.train_minatar --game space_invaders`) + 2 Ollama llama-servers.
nvidia-smi at diagnosis: 5093 MiB used / **2865 MiB free** / 99% util, while V1.1 needs up to 3.3 GB -> a VRAM
spike (Ollama load / AZ replay-eval phase) pushed past 8GB -> CUDA OOM surfaced as a native terminate. It cleared
3 prior ShIC epochs (10/20/30) cleanly -> transient contention, not deterministic.
**Why no data loss:** the trainer checkpoints EVERY epoch (`<tag>_wm_latest.pt` + `epoch_N.pt`) and AUTO-RESUMES
(loads latest.pt unconditionally, `start_epoch = ckpt['epoch']`); `--force` only bypasses run_all_training's
skip-if-complete check, it does NOT reset. So re-running the same command RESUMES from epoch 30 (no rework).
**Fix:** `scripts/wm/train_until_complete.py` (new) -- a crash-resilient driver: (1) VRAM PREFLIGHT gate (waits for
free VRAM >= --min-free-mib before launching; refuses to launch into an oversubscribed GPU -> exit 3); (2) auto-resume
loop (re-launches the SAME resuming command on any non-zero exit, up to --max-retries -- the only way to survive a
native abort, which Python can't catch); (3) sets PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True (anti-fragmentation).
A native abort cannot be caught in-process -> the fix MUST be external. Resume cmd:
`python scripts/wm/train_until_complete.py --model v1_1 --features 41 --run-tag vsn_fr --vsn --forward-regime --max-vram-wait 3600`
**Lesson:** on a single 8GB box, WM training and the games AZ training + Ollama cannot coexist -- serialize the GPU
(pause one) or gate on free VRAM. RWYB-verified: VRAM gate correctly refused at 2865<3500 MiB free.

## 2026-03-18: Undefined SEED in NaN recovery
**Severity:** Critical (runtime crash)
**Files:** `src/wm/v1/v1_1_training/train_world_model.py:678`
**Problem:** `new_seed = SEED + nan_recovery_count * 1000` references undefined `SEED` constant.
Crashes with NameError if nan_frac > 0.5 triggers NaN recovery.
**Fix:** Changed to `new_seed = 42 + nan_recovery_count * 1000` (matches V1.4 pattern).

## 2026-03-18: OOS eval tuple unpacking crash
