# V14.0 Fix Log

## 2026-04-08: VIB bottleneck added on TwoHot path (commit 9973ca0)
**Severity:** HIGH (memorization risk; would have failed f34 tournament identically to V3)

### Bug: Same memorization bug class as V3-clean
**Files:** `world_model.py`, `settings.py`

V14 has two prediction paths:
1. **TwoHot return heads** (V1-compatible): `condition_encoder(WaveNet) → ATME (no-op) → return_head`
2. **Diffusion denoiser**: `(noisy_return, condition) → ReturnDenoiser → predicted_noise`

The TwoHot path is what `validate_world.py` measures with ShIC. It's a direct WaveNet→head path with no bottleneck — same bug class as V3-clean (Pattern I in fix_logs/INDEX.md).

The diffusion path has implicit noise regularization (DDPM trains on noisy returns), but it's a separate path that doesn't protect the TwoHot heads.

**Fix:** Added Variational Information Bottleneck on the TwoHot path:
- VIB_Z_DIM=32 (same as V3, d_model=256)
- VIB_KL_WEIGHT=0.05, annealed via VIB_KL_ANNEAL_EPOCHS=20
- TwoHot return heads now read `z_expand(z)`, diffusion denoiser still reads condition directly (preserves the diffusion innovation)
- Regime head still reads h_seq

**Smoke:** 2,378,598 params, KL=1.58 at init (logvar=-4 → std=0.135 → KL≈mu²/2 ~ 0.5 per dim × 32 dims). VIB grad 20.9, diffusion still fires, both paths backprop independently.

### Architectural rationale
The diffusion path is V14's headline innovation and works well for distribution prediction. The TwoHot path is auxiliary V1-compat infrastructure. By VIB-ing only the TwoHot path, we:
- Preserve the diffusion innovation completely
- Make ShIC validation trustworthy (no memorization shortcut)
- Force the encoder to share useful predictive features via the bottleneck (helps both paths if there's transfer)
