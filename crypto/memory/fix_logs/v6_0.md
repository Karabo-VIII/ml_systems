## 2026-04-07: --clean argparse + clean-variant log_vars IndexError
**Severity:** High (blocked f34 training start)
**Files:** `src/wm/v6/v6_training/train_world_model.py`
**Commits:** 8371b2f, 5f25f97

### Bug 1: --clean flag rejected by main parser (8371b2f)
Same pattern as V3. Pre-parser at line 55-58 consumes `--clean` via `parse_known_args()` to select `TransformerDiscriminatorClean`, but main parser at line 862 rejected the unstripped arg. Added `parser.add_argument("--clean", ...)` no-op.

### Bug 2: log_vars layout mismatch for clean variant (5f25f97)
Epoch-banner logging at line 663-676 hardcoded full JEPA layout `[con, ?, ?, ret_1..ret_4, regime]` (8 elements) and read `_s[3]` for r1 and `_s[3+len(REWARD_HORIZONS)] = _s[7]`. Clean variant has 5-element `[ret_1..ret_4, regime]`. Fix: branch on `_s.numel()` to use clean-layout indices.

### Bug 3: Runner f25 taxonomy mismatch (8371b2f)
Runner declared V6-clean as `[13, 25, 34]`. V6 lives in 41-feat SOTA taxonomy, no f25 slice. Fixed to `[13, 34]`.

### Bug 4: Validation gate false-fail on clean (a85c4f0)
`check_gate()` at line 303-308 applied `contrastive_acc < GATE_CONTRASTIVE_MIN` and `embed_std < GATE_EMBED_STD_MIN`. TransformerDiscriminatorClean has NO JEPA contrastive head and NO VICReg projection — both metrics default to 0 and always fail gate. Fix uses `_pre_args.clean` to skip both checks on the clean variant. Same pattern as V3 Bug 4 and V9 Bug 4.

## 2026-03-17: Huber shortcut + undertraining + checkpoint infra fixes
**Severity:** High (multiple issues)
**Files:** `settings.py` + `train_world_model.py` across all 4 V6 variants

### HIGH-1: DIRECT_RETURN_WEIGHT too high
**Files:** `settings.py` (all V6 variants)
**Problem:** DIRECT_RETURN_WEIGHT=3.0 creates Huber regression shortcut that bypasses
TwoHot discretization bottleneck, accelerating memorization. Same bug as V3/V4.
**Fix:** 3.0 -> 1.0

### HIGH-2: DIVERSITY_STEPS_PER_EPOCH undertraining
**Files:** `settings.py` (all V6 variants)
**Problem:** DIVERSITY_STEPS_PER_EPOCH=300 vs V1.0's 2000. NCL diversity training gets
6.7x less data exposure per epoch.
**Fix:** 300 -> 2000

### HIGH-3: Missing strict=False on load_state_dict
**Files:** `train_world_model.py` (all V6 variants)
**Problem:** model.load_state_dict() and ema_model.load_state_dict() called without
strict=False, causing checkpoint load failures when model schema changes.
**Fix:** Added `strict=False` to model and ema_model load_state_dict calls.

### HIGH-4: shic_decline_count not persisted in checkpoints
**Files:** `train_world_model.py` (all V6 variants)
**Problem:** shic_decline_count initialized to 0 but never saved/loaded from checkpoints.
Training restarts reset the counter, allowing infinite ShIC declines.
**Fix:** Added shic_decline_count to save() signature and state dict, load from checkpoint
in load_latest(), 6-tuple return from load_latest().

### HIGH-5: V6 base missing n_features in save + collision guard in load
**Files:** `train_world_model.py` (V6 base only; V6.1-V6.3 already had these)
**Problem:** V6 base checkpoint didn't save n_features and had no collision guard.
**Fix:** Added n_features to save(), _expected_n_features to ckpt_mgr, collision guard
in load_latest().

### MEDIUM-1: Collision guard used RuntimeError instead of graceful fallback
**Files:** `train_world_model.py` (V6.1, V6.2, V6.3)
**Problem:** Feature mismatch raised RuntimeError, crashing instead of starting fresh.
**Fix:** Changed to print warning + return fresh-start tuple (consistent with V6 base pattern).
