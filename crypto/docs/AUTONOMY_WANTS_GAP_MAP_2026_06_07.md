## 2. TOP missing/weak, ranked by value

1. **EVAL / fitness harness for the autonomy engine itself** — *MISSING (the keystone).* There is no
   `harness/metaop/eval.py` / `fitness.py` / task-solve-rate benchmark (RWYB: all four candidate paths absent).
   The mechanical `verify_cmd` is per-node ground truth, but there is **no aggregate "did the engine solve the task"
   score**. This is the gate that unlocks DGM (#4), AlphaEvolve (#5), and DSPy planner-prompt compilation — none can
   be built without an automated evaluator. *The roadmap's own honest meta-lesson: "the fitness function is the hard
   part, not the mutation engine."*

2. **Real compounding memory (Mem0 / vector)** — *WEAK.* `learnings.py` is TF-IDF over stored objective+lesson
   (`harness/metaop/learnings.py:88`). No embeddings, no extract→update consolidation, no confidence-gated write, no
   surprise-weighting (Titans). With a weak local brain this both forgets nuance and risks hallucinated memories.

3. **Self-evolution beyond Reflexion-lite (DGM skill-archive / AlphaEvolve islands / EvoFSM)** — *MISSING.* `reflect`
   + flat `skill_library` is the whole "improve over cycles" story. The benchmark-before-promote / population-archive
   machinery is design-only in `HARNESS_EVOLUTION_ROADMAP_2026_06_06.md`. Gated on #1 (needs the eval harness).

4. **Robust planner-prompt (DSPy compilation)** — *MISSING / WEAK planner.* The plan node is generic-prompt + one
   structured object (`harness/metaop/graph.py:273` `plan`); the replanner fixes *recovery*, but the *initial* plan
   prompt is hand-written, not compiled/optimized. No DSPy. Gated on #1.

5. **3-loop concurrency made real (meta-loop + project-evolution loop as running programs)** — *WEAK.* Only the
   solver loop is a process; the meta and 3-hourly evolution loops are OVERSEER cadence-prose. The user explicitly
   wants three concurrent loops ("Under /orc: orchestrate, don't self-execute").

6. **Depth+breadth + self-summon-at-cadence as enforced/scored mechanics** — *WEAK.* framer/recaller feed the planner
   (good), but breadth and the 25/50/75% oracle/auditor self-summons are advisory prose, not loop nodes with a
   value-needle score. Pairs with #1.

7. **Goal-bounds pre-flight as code** (budget / value-floor / wall-clock-anchor / stop-conditions checked
   mechanically before a run) — *WEAK / prose.* Currently AUTONOMOUS_RUNNER text, not a gate.

## 4. Honest bottom line

The **mechanics** the user asked for are largely built and verified: a real durable LangGraph loop with a genuine
replanner, a mechanical (unspoofable) verifier, CDAP + mandatory-gate audit, no-prompt/no-stuck control, cross-process
resume, a project-agnostic standalone harness with a drift firewall, a pluggable+installable local brain (LiteLLM +
Ollama, live-verified on this machine), and a working chess agent-learns demonstrator. Since the N1 snapshot, **N3
(replanner) and N4 (LiteLLM brain) have closed** — the two biggest items N1 flagged.

The **"thinks/evolves for itself" core remains weak in four integration-not-reinvention places**, all gated on one
keystone: **(1) an EVAL/fitness harness** (missing — unlocks the rest), then **(2) real Mem0 memory** (TF-IDF today),
**(3) self-evolution (DGM/AlphaEvolve/EvoFSM)** (doc-only), and **(4) a compiled planner prompt + cascade router**
(hand-written today). The depth+breadth + cadence-self-summon machinery is wired as *advisory context* but not *scored
or enforced*. These are the next build veins.
